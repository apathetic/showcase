    About

    CHUSY is a framework modeled after SUSY, its goal is to be like SUSY, but not opaque and confusing. It makes prototyping and maintenance of multi-breakpoint liquid grids fast, easy, and fun!

    There are two flavors: "Original" and "Flexible Gutter", which are very much alike. While both are fully fluid, there are differences is how the columns and gutters work. Here is a brief overview:

    Original

    How it works: Orginal takes base width, column width, gutter-width and column inner padding to calculate precise percentages for each element.

    • Pro: User can define Column padding and gutter width with a specific number, which will be converted to a percentage automatically. It gives user more options to control the look of the grid.
    • Cons: Column padding and gutter are fluid. It can't create fixed padding and gutter. It requires some level of calculation to set up the grid.

    If you have any question about the Original, please contact Choshun Snyder

    Flexible Gutter

    How it works: Flexible gutter gets the column width by taking 100% divided by the total # columns. It uses column inner padding to create a gutter.

    • Pro: User can define gutters with a fixed number or percentage to create fixed or fluid gutter look. No calculation for setting up the grid, simply enter the total column.
    • Cons: Since the column uses column inner padding to fake the gutter, you may need extra markup if more padding is needed between the content and its parent column

    If you have any question about the Flexible Gutter variant, please contact Miranda Li

    Installation

    Original

    Copy and paste _base.scss and _mixins.scss into a working compass project (http://compass-style.org/install/)

    Reference _base and _mixins in needed scss files with:

    {% highlight css %} @import "base"; @import "mixins"; {% endhighlight %}

    Done!

    Flexible gutter

    Copy and paste _base.scss and _mixins-flexible-gutter.scss into a working compass project (http://compass-style.org/install/)

    Reference _base and _mixins-flexible-gutter in needed scss files with:

    {% highlight css %} @import "base"; @import "mixins-flexible-gutter"; {% endhighlight %}

    Done!

    Options

    Making breakpoints

    In _base.scss there are breakpoint variables that are defined by browser viewport widths.

    For example:

    {% highlight sass %}$small-screen: small-screen 0 599px;{% endhighlight %}

    $small-screen is a breakpoint variable that defines the threshold width of 0 to 599px.

    This is the ONLY place you'll need to define breakpoints for the whole project.

    You did it! good job

    Making grids

    At the top of _mixins is the mixin at-breakpoint($viewport).

    For each breakpoint there is an associated grid configuration. This creates the liquid grid that has unique column, gutter, and grid padding in percent.

    In the case of using our previous breakpoint $small-screen we define the grid:

    {% highlight sass %} // if the first value of the variable $small-screen (it's value being: "small-screen 0 599px") is small screen... @if nth($viewport, 1) == small-screen { // at base ideal width of 400... $base-width: 400px; // the columns will be this ideal px width (which in turn produce the number of liquid columns) $column-width: 43.3333333333333px; // with the gutters of the columns being 20px $gutter-width: 20px; // the grid padding of the whole grid on the left and right will be 20px $grid-padding: 20px; // don't worry about me $active-width: $base-width - (2 * $grid-padding); // another thing that is derived from previous values $inner-padding: 100 / ($active-width / $grid-padding) * 1%; } {% endhighlight %}

    This creates a 6 column liquid grid for the breakpoint $small-screen.

    This is the ONLY place you define grids.

    NOTE: It is important that you calculate the values as precise as possible (ie 43px WILL look way different than 43.33333333333333333333px). Don't trust the designs, plug in the cols/gutters/grid padding and make sure it equals the defined ideal width.

    Huzzah!

    Using the Responsive Grid

    Original

    We have breakpoints from _base, and unique liquid grids for each breakpoint defined in at-breakpoint() in _mixins, how do we use these in our scss files?

    With the mixin span-columns() inside the at-breakpoint() mixin of course!

    Example 1
    {% highlight sass %} #wrapper { @include at-breakpoint($small-screen) { #header { @include span-columns(3); } } } {% endhighlight %}

    At viewport $small-screen (0-599px defined in _base), the #header will span 3 columns, in this case 3 columns will be ~50%, cause the grid ends up being a 6 col grid (see step 3)

    Example
    {% highlight sass %} #wrapper { @include at-breakpoint($large-screen) { #header { @include span-columns(3); } } } {% endhighlight %}

    At viewport $large-screen (1000px up as defined in _base) the same exact #header div will be at ~25%, 'cause that grid ends up being 12 columns as defined in _mixins.

    That's it!

    The rest of the mixins are just commented frosting on the delicious sponge cake that is CHUSY.

    Flexible Gutter

    In the flexible gutter version of chusy, `column width` is 100% divided by the total columns. The gutter is created by the padding on the columns. You can specify gutter width in _base.scss `$gutter-width`. You can define total number of columns in the break point mixin: {% highlight sass %} //in _base.scss $gutter-width: 30px; // it creates 15px padding left and right on a column // in _mixins-flexible-gutter.scss // if the first value of the variable $small-screen (it's value being: "small-screen 0 599px") is small screen... @if nth($viewport, 1) == small-screen { // defines total columns $total-columns: 6; // calculate each column's width $column-width: (100 /$total-columns) * 1%; // defines the margin between content and the browser border. $page-margin: 3%; } {% endhighlight %}

    Define the column width with span-columns();

    Example
    {% highlight sass %} .side-content { // .side-content will take up 3 columns as its width: 3 * 100% / 12 = 25% @include span-columns(3); // .side-content will take up 6 columns as its width: 6 * 100% / 6 = 100% @include at-breakpoint($small-screen) { @include span-columns(6); } } {% endhighlight %}
